## 核心调研流程

### 第一阶段：快速扫描 (15-30 分钟/篇)
1. **标题与摘要** - 核心问题、主要贡献
2. **图表浏览** - 架构图、实验结果图
3. **结论部分** - 主要发现、局限性
4. **相关工作** - 定位论文在领域中的位置

### 第二阶段：深度分析 (1-2 小时/篇)
1. **问题定义** - 作者想解决什么？为什么重要？
2. **方法拆解** - 核心创新点、技术路线
3. **数学表达** - 公式推导、符号说明
4. **实验设计** - 数据集、基线、评估指标
5. **代码实现** - 关键算法、数据流

### 第三阶段：批判性评估 (30 分钟/篇)
1. **假设检验** - 方法依赖哪些假设？
2. **局限性** - 作者承认的 + 自己发现的
3. **可复现性** - 代码/数据是否开源？
4. **迁移价值** - 能否用到我们的场景？

## 论文选择标准

### 经典性 (权重 30%)
- 被引量
- 领域影响力
- 是否开创性方向

### 实用性 (权重 30%)
- 代码是否开源
- 是否可复现
- 工程迁移价值

### 前沿性 (权重 25%)
- 发表时间
- 是否代表新方向
- 与当前研究的相关性

### 多样性 (权重 15%)
- 不同方法类型
- 不同任务场景
- 不同机器人平台

## 知识管理

### 记录位置
- 深度分析 → `research/papers/[paper-name].md`
- 代码笔记 → `research/code/[project-name]/`
- 对比总结 → `research/comparisons/[topic].md`

### 标签体系
- `#VLA` - Vision-Language-Action
- `#imitation-learning` - 模仿学习
- `#RL` - 强化学习
- `#diffusion` - 扩散模型
- `#transformer` - Transformer 架构
- `#real-world` - 真实机器人实验

## 持续改进

### 每次调研后反思
1. 哪些分析最有价值？
2. 哪些信息遗漏了？
3. 下次如何改进流程？
4. 发现了什么新模式？

### 更新频率
- 每完成 5 篇论文 → 更新框架
- 每月回顾 → 整理知识图谱
- 每季度 → 输出综述报告
